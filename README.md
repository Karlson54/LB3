# Електронна система торгівлі (E-Commerce)

## Опис програми

Ця програма є простим прикладом електронної системи торгівлі (e-commerce). Основна її функціональність включає у себе додавання товарів до кошика, оформлення замовлення та взаємодію з замовленнями.

## Опис роботи програми

### Клас `Product`

Клас `Product` представляє товар і має наступні атрибути:

- `id` (ідентифікатор товару)
- `name` (назва товару)
- `price` (ціна товару)

### Клас `Cart`

Клас `Cart` представляє кошик для товарів. Має такі методи:

- `addProduct(Product product)`: Додає товар до кошика.
- `removeProduct(Product product)`: Видаляє товар з кошика.
- `getProducts()`: Повертає список товарів у кошику.

### Клас `Order`

Клас `Order` представляє замовлення і має такі атрибути:

- `orderId` (ідентифікатор замовлення)
- `products` (список товарів у замовленні)
- `status` (стан замовлення)

Методи включають:

- `getOrderId()`: Повертає ідентифікатор замовлення.
- `getProducts()`: Повертає список товарів у замовленні.
- `getStatus()`: Повертає стан замовлення.
- `setStatus(String newStatus)`: Встановлює новий стан замовлення.

### Клас `ECommerceSystem`

Клас `ECommerceSystem` містить деякі статичні методи для взаємодії з системою електронної торгівлі:

- `placeOrder(Cart cart)`: Додає додаткові товари до кошика, якщо він не пустий.
- `getOrderStatus(Order order)`: Повертає стан замовлення.
- `updateOrderStatus(Order order, String newStatus)`: Оновлює стан замовлення.

## Засіб взаємодії

Основні методи:

- `placeOrder(Cart cart)`: Додає додаткові товари до кошика та оформлює замовлення.
- `getOrderStatus(Order order)`: Повертає стан замовлення.
- `updateOrderStatus(Order order, String newStatus)`: Оновлює стан існуючого замовлення.

## Five Why's принцип

1. **Чому була створена програма?**
   - Для моделювання базового процесу електронної торгівлі, де користувач може додавати товари до кошика, робити замовлення та взаємодіяти зі статусами замовлень.

2. **Чому потрібен клас `Cart`?**
   - `Cart` використовується для збереження товарів, які користувач додав до свого кошика, щоб легко взаємодіяти з ними.

3. **Чому використовується клас `Order`?**
   - `Order` дозволяє створювати та відстежувати замовлення, їх статуси та вміст.

4. **Чому є метод `ECommerceSystem.placeOrder`?**
   - `ECommerceSystem.placeOrder` використовується для автоматичного додавання додаткових товарів до кошика при оформленні замовлення.

5. **Чому використовуються тести?**
   - Тести використовуються для перевірки правильності роботи окремих функцій та забезпечення надійності програми.

## Тести

Тести включають:

1. **testAddProductToCart**: Перевіряє додавання товару до кошика та правильність інформації про товар.
2. **testRemoveProductFromCart**: Перевіряє видалення товару з кошика.
3. **testPlaceOrder**: Перевіряє, чи вірно додаються додаткові товари при оформленні замовлення.
4. **testGetOrderStatus**: Перевіряє правильність отримання статусу замовлення.
